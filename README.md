# ResearchManager
Researchmanagerは，作者（seytwo）が自身の好みに合わせて開発した，研究管理アプリです．

## deck

### 概要
現時点（2020/4/12）で，ResearchManagerには「deck」の機能があります．
deckはメモ・アイデア・タスクを「気軽に」「適当に」投稿できるアプリです．

研究管理方法の一つに「日記を書くこと」があります．
しかし，作者はブログなどのSNSが続いたことがありません．
長い文章を，毎日，それなりに形式を整えて書くのが面倒だからです．

一方で，twitterだけは10年以上続いています．
短い文章で「気軽に」「適当に」投稿できるからです．
deckは，長く続けられるアプリとなるように，twitterを参考に開発しました．

### 機能
deckでは以下のことができます．

- 「チケット」の投稿と確認  
投稿の単位を「チケット」と呼びます．
日々のメモ・アイデア・タスク・進捗などをtwitterのように投稿し，確認できます．

- 「メモ」「タスク」の分類  
チケットは「メモ」と「タスク」に分けられます．
メモは適当なアイデアやタスクの進捗，タスクはtodoを投稿するときに使います．
タスクには進捗状況を設定できます．

- 「カラム」でのチケット一覧の表示  
チケット一覧は「カラム」に表示されます．
カラムにはいくつか種類があり，目的に応じて使い分けます．

- チケット間の紐づけ  
チケットには「親チケット」を設定できます．
これにより，「タスクの進捗メモ」や「アイデアへの追記」ができます．

- 未完了タスク一覧の確認（タスクカラム）  
チケット一覧から未完了のタスクのみを抽出し，確認できます．

- タスクの進捗の確認（タスク進捗カラム）  
タスクに紐づく進捗や小タスクを確認できます．

- ローカル環境での動作  
研究内容はネット上に挙げたくないので．

### 環境構築
私の場合なので，gitやtortoise gitなどは，それぞれが使いやすい方法で大丈夫です．

#### インストール
1. visual studio code(vscode)をインストール
2. node.jsをインストール
3. gitをインストール
4. tortoise gitをインストール

#### リポジトリのクローン
1. https://github.com/seytwo/ResearchManager → [Clone or download] → リポジトリのurlを取得
2. 任意の場所（例えば，デスクトップ)で右クリック → [Gitクローン（複製)] → 1.のurlを指定 → OK

#### node.js/electronの設定
1. vscodeを開く
2. ファイル → フォルダを開く → ローカルに保存したリポジトリを指定
3. 表示 → ターミナル
4. npm init -y  
（package.jsonを作成）
5. npm install electron  
（package-lock.json, node_modulesを作成）
6. npm install electron-packager

#### vscodeで実行する方法
1. electron ./src/index.js  

#### exe/appファイルから実行する方法
1. npx electron-packager ./src ResearchManager --platform=(win32/darwin) --arch=x64 --overwrite  
（(win32/darwin)はどちらかを選択；ResearchManager-(win32/darwin)-x64を作成）
2. [ResearchManager/ResearchManager-(win32/darwin)-x64/resources]に[ResearchManager/data]をコピー
3. [ResearchManager/ResearchManager-(win32/darwin)-x64/ResearchManager.exe]を実行

### 使い方
- アプリの起動  
アプリを起動してください．
デフォルトでは以下の画面が表示されます．
<img width="1080" alt="20200412_3" src="https://user-images.githubusercontent.com/32482617/79065214-abd04a00-7ce9-11ea-9c99-332da74d6228.PNG">

- 開発用ログ  
右側は開発用のログです．
エラーが起きたらすぐに分かるので，小さく表示しておくことを推奨します．

- メニューバー  
左側のボタン（🏠，💼）で，カラムが開きます．
<img width="1080" alt="20200412_4" src="https://user-images.githubusercontent.com/32482617/79065231-d3bfad80-7ce9-11ea-8c50-8c2dc290b691.PNG">

- ホームカラム  
メニューバーの「ホームボタン（🏠）」をクリックすると「ホームカラム」が開きます．
ホームカラムは「よく使うカラム」の位置付けです．
すべてのチケットが表示されるので，一覧をぼーっと眺めて，
最近やったことの確認，アイデアへの追記，タスクの追加をするのが良いかと思います．

- チケットボックス  
チケットは「チケットボックス」に内容が表示されます．
ユーザ情報，各種ボタン，チケット情報が表示されます．
（ユーザ情報はまだ設定できません．）

- チケット追加ボックス  
カラムの上部（ヘッダ；水色背景）にあるのは「チケット追加ボックス」です．
「text」に内容を書いて，「ctrl + enter」でチケットを投稿します．
必要に応じで，「title」に題名を書いてください．
また，チケットの種類は，右側にある「チケット種類選択リスト（[💬▼]）」，
または，その下にある「メモボタン（💬）」「タスクボタン（💼）」で選択してください．

- 親チケット  
「親チケット」はチケット追加ボックスの「parent」に設定します．
親チケットのボックスの「メモボタン（💬）」「タスクボタン（💼）」をクリックすると，
チケット追加ボックスの「parent」に親チケットのIDが設定されます．
このとき，チケット追加ボックスの種類が，それぞれメモとタスクに変更されます．

- チケット内容の修正  
チケットの内容はすぐに修正できます．
なので，「気軽に」「適当に」投稿してください．
チケットボックスの内容を直接修正し，「ctrl + s」で保存してください．
保存すると，ユーザ名の横にある日時が更新されます．

- ファイルの添付  
投稿されたチケットボックスにはファイルを添付できます．
（チケット追加ボックスにあるべきですが，現時点では投稿後に添付します．）
「添付ボックス(📎)」をクリックして添付してください．
（ただし，アプリ上からファイルの閲覧はできません．直接フォルダに閲覧する必要があります．）

- タスクの進捗状況リスト  
タスクの進捗状態は「進捗状況リスト」で設定します．
「S(start)」が未着手状態，「-」が着手状態，「E(end)」が完了状態を意味します．

- タスク進捗カラム  
「タスク進捗ボタン（📓）」をクリックすると，「タスク進捗カラム」が表示されます．
タスク進捗カラムでは，該当のタスクに紐づくチケットの一覧が表示されます．
また，ヘッダには該当のタスクも表示されます．
指定がない場合，チケット追加ボックスの親チケットには該当のタスクが自動で設定されます．
特定のタスクのメモやアイデアを連続で投稿するときには，このカラムで作業をするのが良いと思います．
<img width="1080" alt="20200412_5" src="https://user-images.githubusercontent.com/32482617/79065250-fc47a780-7ce9-11ea-8c0a-e2f55ed85807.PNG">

- タスクカラム  
メニューバーの「タスクボタン（💼）」をクリックすると「タスクカラム」が表示されます．
タスクカラムには未完了のタスクの一覧が表示されます．
ホームカラムにはすべてのチケットが表示されるため，タスクが埋もれてしまいます．
残タスクを確認→特定のタスクの進捗カラムを表示→そのタスクを実施，の流れで使うのが良いと思います．
<img width="1080" alt="20200412_6" src="https://user-images.githubusercontent.com/32482617/79065761-aa088580-7ced-11ea-9a4c-52d3785c80ba.PNG">

- 生データ  
vscodeからの実行であれば[ResearchManager\data\ticket]，
exe/appからの実行であれば[ResearchManager\ResearchManager-win32-x64\resources\data\ticket]
にあります．
データをリセットするときには，ここから削除してください．
